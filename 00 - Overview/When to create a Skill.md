# When to create a Skill

> ไม่ใช่ทุกงานควรเป็น skill — ใช้เกณฑ์ด้านล่างประกอบการตัดสินใจ

## ✅ ควรสร้าง skill เมื่อ...

- งานนั้นจะถูก **ทำซ้ำ** หลายครั้งในรูปแบบเดิม
- มี **ขั้นตอนเฉพาะ** ที่ Claude มักทำผิดถ้าไม่บอก
- ต้องใช้ **เครื่องมือ/script** เดิม ๆ
- ต้องการ **บังคับ output format** ที่แน่นอน
- มี **domain knowledge** ที่ไม่ได้อยู่ในข้อมูลทั่วไป

## ❌ ไม่ควรสร้าง skill เมื่อ...

- งานทำครั้งเดียวแล้วจบ
- คำตอบเปลี่ยนตามบริบทมากเกินไป
- เป็นความรู้ทั่วไปที่ Claude รู้อยู่แล้ว
- ยังไม่ชัดเจนว่าต้องการอะไร — ให้ทดลองทำธรรมดาก่อน แล้วค่อย "extract" ออกมาเป็น skill

## สัญญาณว่า "ถึงเวลา" สร้าง skill

1. ผู้ใช้บอกว่า "ทำแบบนี้ทุกครั้ง..."
2. Claude ทำผิดในจุดเดิม ๆ
3. ต้องอธิบายขั้นตอนเดิมซ้ำหลายครั้ง
4. งานนั้นมี output ที่ต้อง verify ทุกครั้ง (test cases ได้)
