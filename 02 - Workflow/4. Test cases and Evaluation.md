# Step 4 — Test cases and Evaluation

> เขียน 2-3 test prompts ที่ user จริง ๆ น่าจะพิมพ์ แล้วลองรันทั้ง with-skill และ without

## เขียน test prompts

ใส่ในไฟล์ `evals/evals.json`:

```json
{
  "skill_name": "my-skill",
  "evals": [
    {
      "id": 1,
      "prompt": "ขอสรุปไฟล์ PDF ที่แนบมาเป็น bullet points",
      "expected_output": "สรุปเป็น bullet ครอบคลุมหัวข้อหลัก",
      "files": ["sample.pdf"]
    }
  ]
}
```

## รัน 2 condition คู่กัน

ในการรันแต่ละ test ต้องรัน **คู่** เสมอ:

- **with_skill** — มี skill
- **without_skill (baseline)** — ไม่มี skill

นำผลทั้ง 2 มาเปรียบเทียบ

## เก็บผล

```
my-skill-workspace/
└── iteration-1/
    ├── eval-0/
    │   ├── with_skill/outputs/
    │   └── without_skill/outputs/
    └── eval-1/
        └── ...
```

## Quantitative + Qualitative

- **Qualitative** — เปิดดูเอง ผู้ใช้ให้ feedback
- **Quantitative** — เขียน assertions (สิ่งที่ verify ได้ด้วย code)

ตัวอย่าง assertion:
- "output มี header 'Summary'"
- "ไฟล์ที่ generate มี extension .pdf"
- "จำนวน bullet >= 3"

## ใช้ eval-viewer

`skill-creator` มี `generate_review.py` สำหรับเปิด viewer ให้ผู้ใช้ review
