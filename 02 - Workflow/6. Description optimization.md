# Step 6 — Description Optimization

> หลังจาก skill เสร็จ — optimize ให้ trigger แม่นที่สุด

## ทำไมต้อง optimize

`description` คือ **ตัวเดียว** ที่ Claude ใช้ตัดสินว่าจะเรียก skill หรือไม่ — ถ้าเขียนไม่ดี skill จะไม่ถูกเรียกใช้เลย

## ขั้นตอน

### 1. สร้าง trigger eval

20 query — ครึ่งหนึ่ง should_trigger ครึ่งหนึ่ง should_not_trigger:

```json
[
  {"query": "ขอสรุป PDF ในไฟล์แนบให้ทีหน่อย", "should_trigger": true},
  {"query": "เขียน function หาค่า fibonacci", "should_trigger": false}
]
```

**Tip**:
- ใส่ realistic context (ชื่อไฟล์, ชื่อ column, situation)
- should-not-trigger ต้อง "near-miss" ไม่ใช่กระจัดกระจาย

### 2. รัน optimization loop

```bash
python -m scripts.run_loop \
  --eval-set <eval.json> \
  --skill-path <skill> \
  --model <model-id> \
  --max-iterations 5 \
  --verbose
```

Script จะ:
- แบ่ง 60% train / 40% test
- รันแต่ละ query 3 ครั้ง (เพื่อค่า trigger rate)
- เรียก Claude ปรับ description
- ทำซ้ำ 5 รอบ
- เลือก best_description ตาม test score

### 3. apply ผลลัพธ์

นำ `best_description` ใส่ใน frontmatter
