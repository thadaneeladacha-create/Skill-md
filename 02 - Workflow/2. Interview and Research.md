# Step 2 — Interview and Research

> ก่อนเขียน skill ทำการบ้านให้พอ จะได้ไม่ต้องวนซ้ำหลายรอบ

## ถามอย่างมีเป้าหมาย

ถามเฉพาะเรื่องที่ส่งผลต่อการออกแบบ skill:

- Input format มีกี่แบบ?
- Output ต้องดูเป็นแบบไหน?
- Edge cases ที่เคยเจอ?
- ใช้ library อะไรเป็นมาตรฐาน?

อย่าถามท่วม — รวมหลายคำถามใน 1 รอบ

## วิจัยเพิ่ม

- ลองค้น docs ของ tool/library ที่จะใช้
- หา skill อื่นที่คล้ายกัน เพื่อดู pattern
- ใช้ MCP search ถ้ามี

## เป้าหมาย

จบขั้นนี้ต้องมี:
- ✅ requirement ชัด
- ✅ ตัวอย่างไฟล์ test (1-2 ตัวอย่าง)
- ✅ พอใจกับ scope ของ skill
