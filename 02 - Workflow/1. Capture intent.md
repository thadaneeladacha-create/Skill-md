# Step 1 — Capture Intent

> ก่อนเขียนโค้ดใด ๆ ต้องเข้าใจให้ชัดว่า skill นี้ทำอะไรและ trigger เมื่อไหร่

## 4 คำถามหลัก

1. **Skill นี้ทำให้ Claude ทำอะไรได้?**
2. **ควร trigger เมื่อไหร่?** (วลี/บริบทของ user)
3. **Output ที่คาดหวังคืออะไร?** (รูปแบบไฟล์ โครงสร้าง)
4. **ต้องตั้ง test cases มั้ย?** — ถ้า output verify ได้ แนะนำให้ตั้ง

## ดึง intent จากบทสนทนา

ถ้าผู้ใช้พูดว่า "ให้ทำแบบนี้เป็น skill" — ดูประวัติบทสนทนาก่อน:
- ใช้ tool อะไร
- ลำดับขั้นตอน
- การ correction ของ user
- รูปแบบ input/output

แล้วค่อยถามเฉพาะส่วนที่ขาด

## สิ่งที่ต้องชัดเจน

- ✅ เกณฑ์สำเร็จ
- ✅ Edge cases ที่อาจเจอ
- ✅ ตัวอย่างไฟล์/ข้อมูล input
- ✅ Dependencies (library, tool)
