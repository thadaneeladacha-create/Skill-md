# Step 3 — Write SKILL.md (draft)

> เขียน draft แรก ไม่ต้องสมบูรณ์แบบ — เน้นโครงสร้างก่อน ค่อย iterate

## โครงสร้างที่ควรมี

1. **Frontmatter** — name + description (ดู [[Description writing]])
2. **Overview สั้น ๆ** — ทำอะไร, สำหรับใคร
3. **Workflow** — ขั้นตอนตามลำดับ
4. **Best practices / กฎสำคัญ** — สิ่งที่ห้ามผิด
5. **References** — link ไป `references/` ถ้ามี
6. **ตัวอย่างการใช้งาน** — 1-2 ตัวอย่าง

ดูเพิ่มที่ [[SKILL.md structure]]

## เคล็ดลับการเขียน

### ใช้ imperative form

```markdown
❌ "User ควรจะตรวจสอบ missing values"
✅ "ตรวจสอบ missing values ก่อน train"
```

### อธิบาย why

```markdown
❌ "ALWAYS ใช้ stratify=y"
✅ "ใช้ stratify=y เพื่อให้สัดส่วนของ class ใน train/test
   เท่ากัน — ป้องกัน metric เพี้ยนเมื่อ class imbalance"
```

ดูเพิ่มที่ [[Writing style]]

## หลังเขียน draft

- ✅ ทบทวนความยาว — ตัดส่วนที่ไม่จำเป็น
- ✅ ตรวจว่า description ตรงกับเนื้อหา
- ✅ ลองอ่านเหมือนเป็น Claude — เข้าใจมั้ย?
